# Deploy

The Mac mini runs the coordinator, web UI, Worker A, and Cloudflare Tunnel. Worker B runs on the adesso MacBook and connects to the public HTTPS coordinator URL.
The installed web service listens on `:3000`, starts Next.js internally on `:3001`, and proxies `/api/*`, `/openapi.json`, `/docs`, `/redoc`, and `/healthz` to the coordinator on `:8000`.
Deployment preflight checks can confirm CLI commands, API credentials, local model services, named tunnel config, and launchd wiring are present, but they do not run model prompts or prove unattended CLI auth. Mac mini preflight verifies the named `cloudflared` launchd service uses `~/.cloudflared/config.yml` and runs the same tunnel declared in that config. API credential checks count real keys from the current shell or from the installed worker launchd environment and ignore placeholder values. Use acceptance runs after enabling models to prove the actual routing path. Grok CLI is only counted when `grok --help` advertises noninteractive `-p`/`--prompt` mode; otherwise set `XAI_API_KEY` to use the xAI API fallback. Set `GEMINI_API_KEY` to use the Gemini API-backed `gemini-2.5-flash` adapter without depending on Gemini CLI auth.
Worker preflight also reports the configured `allowed_models` pin and whether the installed worker launchd service contains `GEMINI_API_KEY` and `XAI_API_KEY`. Keep the pin set to `codex-gpt-5.5` until additional auth is proved. After `GEMINI_API_KEY` is configured on both workers, use `ALLOWED_MODELS=codex-gpt-5.5,gemini-2.5-flash`; the consolidated final acceptance sequence uses `WORKER_REQUIRED_CAPABILITIES=codex-gpt-5.5,gemini-2.5-flash` by default and its readiness gate runs deploy preflight with launchd API-key requirements for API-backed final models. If a launchd API-key check warns or fails for a key you expect, rerun `make install-worker` with that key in the environment because launchd does not inherit later shell changes.

## Mac mini

1. Install dependencies with `make bootstrap`.
2. Run `make deploy-preflight DEPLOY_ROLE=mac-mini` and resolve any `FAIL` lines.
3. Prefer `make setup-named-tunnel TUNNEL_NAME=dialectical TUNNEL_HOSTNAME=debate.<your-domain>` for the login/create/install/preflight/status sequence. The Makefile target stops the quick tunnel by default only after deploy preflight and endpoint status verify the named hostname. If `SETUP_NAMED_TUNNEL_FLAGS` includes `--skip-status` or `--skip-preflight`, the helper refuses to refresh handoff bundles unless `--allow-unverified-handoff` is also set, and it refuses to stop the quick tunnel because the named endpoint and launchd preflight have not both been verified. Set `STOP_QUICK_TUNNEL_AFTER_VERIFY=0` only if you intentionally need to keep the temporary tunnel alive.
4. If installing manually, run `make install-tunnel TUNNEL_NAME=dialectical TUNNEL_HOSTNAME=debate.<your-domain> CLOUDFLARED_CREDENTIALS=$HOME/.cloudflared/<tunnel-id>.json`. `TUNNEL_NAME` must be a real Cloudflare tunnel name or UUID, `TUNNEL_HOSTNAME` must be a DNS hostname, not a URL or `trycloudflare.com` quick tunnel host, and the credentials JSON must contain `AccountTag`, UUID-shaped `TunnelID`, and `TunnelSecret`. After the named tunnel is verified, run `make stop-quick-tunnel` so final status no longer uses the temporary quick-tunnel URL.
5. Build the web UI with `pnpm --dir web build`.
6. Install coordinator and web launchd services with `make install-services`.
7. Read the user token from the coordinator launch log on first boot if you did not set `DIALECTICAL_USER_TOKEN`: `tail -n 50 /tmp/dialectical-coordinator.out.log`.
8. Register and start Worker A with `make install-worker COORDINATOR_URL=http://localhost:8000 WORKER_NAME=mac-mini`. Set `DIALECTICAL_ALLOWED_MODELS=codex-gpt-5.5` first if installed Claude/Gemini CLIs should stay hidden until unattended auth is proved. Set `GEMINI_API_KEY` or `XAI_API_KEY` in the same environment if the launchd worker should use those API-backed adapters.
8. Rerun `make deploy-preflight DEPLOY_ROLE=both PREFLIGHT_FLAGS="--require-installed-services --require-registered-worker"` and resolve any `FAIL` lines. If it reports an old `user_token` in the worker config, rerun with `PREFLIGHT_FLAGS="--repair-worker-config --require-registered-worker"`.
9. Check the live service state with `make status STATUS_FLAGS=--check-endpoints`; endpoint-check failures make the command exit nonzero.
10. Generate handoff bundles with `make handoff-bundles PUBLIC_URL=https://debate.<your-domain>`.
11. Confirm the public hostname with `make acceptance COORDINATOR_URL=https://debate.<your-domain> EXPECTED_WORKERS=1 EXPECTED_WORKER_NAMES=mac-mini`. The target prompts for the user token with terminal echo disabled unless `USER_TOKEN` is already set in the environment. Worker B onboarding and URL-switch helpers also verify the token-free `/api/backends/status` endpoint before prompting for a token or changing Worker B config.

## adesso MacBook

1. Install dependencies with `make bootstrap`.
2. Run `make deploy-preflight DEPLOY_ROLE=worker` and resolve any `FAIL` lines.
3. Register using `make install-worker COORDINATOR_URL=https://debate.<your-domain> WORKER_NAME=adesso-mbp`. For unattended setup, set `DIALECTICAL_USER_TOKEN` or `USER_TOKEN` in the environment. Set `DIALECTICAL_ALLOWED_MODELS=codex-gpt-5.5` until additional auth is proved; set `GEMINI_API_KEY` and use `DIALECTICAL_ALLOWED_MODELS=codex-gpt-5.5,gemini-2.5-flash` when the Gemini API should provide the second real model. Set `XAI_API_KEY` in the same environment when using the xAI API fallback. `make install-worker` copies present API keys into the installed launchd service. The generated Worker B onboarding script applies the `codex-gpt-5.5` pin by default and verifies the public worker status advertises it; pass `ALLOWED_MODELS=` only when intentionally clearing the pin. If no healthy adapter matches the allowlist, registration stops before saving config or updating the coordinator.
4. `make install-worker` installs and starts the user launchd worker service.
5. Rerun `make deploy-preflight DEPLOY_ROLE=worker PREFLIGHT_FLAGS="--require-registered-worker --require-installed-services"` and resolve any `FAIL` lines. If it reports an old `user_token` in the worker config, rerun with `PREFLIGHT_FLAGS="--repair-worker-config --require-registered-worker"`.
6. If Worker B was registered against a `trycloudflare.com` quick tunnel before the named hostname existed, use the Worker B bundle's `switch_worker_b_url.sh` with `NEW_COORDINATOR_URL=https://debate.<your-domain>`. The helper verifies the token-free public endpoint before changing config, restarts launchd, runs worker preflight, and verifies public visibility. Omit `ALLOWED_MODELS` to preserve the current allowlist; pass `ALLOWED_MODELS=` to clear it.

## Two-Worker Acceptance

Once Worker B is registered from the MacBook, run final production acceptance
from the Mac mini so the reports land under `/private/tmp` where strict status
reads them. If a phase is run from another machine, copy its JSON report to the
same `/private/tmp/dialectical-acceptance-*.json` path on the Mac mini before
the next phase and before final strict status.

```bash
make acceptance COORDINATOR_URL=https://debate.<your-domain> EXPECTED_WORKERS=2 EXPECTED_WORKER_NAMES=mac-mini,adesso-mbp
```

The target prompts for the user token with terminal echo disabled unless `USER_TOKEN` is already set in the environment. The run includes structured public archive API and web-home evidence, structured worker-status payload counts and rows, structured debate lifecycle evidence for create/skeleton/role-override/timing/persistence checks, structured auth-boundary evidence for public-read/auth-write rejection checks, structured authenticated settings evidence that verifies per-model spend/cap fields and restores the original cap map, plus auth-gated web prompt, token-flow source-contract, post-unlock source-surface, debate regenerate/history action-control source-contract, browser SSE streaming-client source-contract checks, structured SSE event/token evidence, structured online/offline worker rows with IDs, generated/regenerated node metadata, worker names, model ID fields, authenticated regenerate-request job evidence, named HTTPS URL-class proof, and the exact regeneration model switch tied to archived/active history. Use `REQUIRE_WORKERS_IN_TREE=1 ACCEPTANCE_REQUIRE_NAMED_HTTPS=1` on the first run to prove both workers generated debate nodes through the final hostname, and keep `REQUIRE_DIFFERENT_REGEN_MODEL=1` for final production proof after at least two real models have proved unattended auth and are enabled for routing. A provisional one-model run can set `REQUIRE_DIFFERENT_REGEN_MODEL=0`, but `make status` will report it as incomplete final proof. Then power off or sleep the MacBook, wait at least 90 seconds, and rerun with `EXPECTED_WORKERS=1 EXPECTED_WORKER_NAMES=mac-mini EXPECTED_OFFLINE_WORKER_NAMES=adesso-mbp ACCEPTANCE_REQUIRE_NAMED_HTTPS=1`; this proves Worker B was registered and is now offline instead of never installed. Wake the MacBook and rerun with `EXPECTED_WORKERS=2 EXPECTED_WORKER_NAMES=mac-mini,adesso-mbp REQUIRE_WORKERS_IN_TREE=1 ACCEPTANCE_REQUIRE_NAMED_HTTPS=1`.

The generated Worker B bundle includes `production_acceptance.sh`, which wraps these three phases with `MODE=two-worker`, `MODE=failover-one-worker`, and `MODE=rejoin-two-worker`. The helper waits for the expected public worker state before each run, including Worker B `offline` for the failover phase, and requires a passed two-worker report before failover plus a passed failover report before rejoin. It parses those prior reports and rejects stale phase, URL, web/SSE, named-HTTPS, or different-model proof before prompting for the user token, then validates the report from the current run before printing success.
For final strict status, run all three helper phases from the Mac mini or copy each report back to the same `/private/tmp` path on the Mac mini. Nonstandard report directories are rehearsal-only and require the strict-validation rehearsal skip flags.
It rejects placeholder, non-HTTPS, local, and `trycloudflare.com` quick-tunnel URLs by default and passes `ACCEPTANCE_REQUIRE_NAMED_HTTPS=1` into `make acceptance`; set `ALLOW_QUICK_TUNNEL_ACCEPTANCE=1` only for an explicit provisional quick-tunnel smoke run. Final production acceptance must use the named Cloudflare hostname.
The standalone Worker B acceptance helper requires each online worker to advertise `codex-gpt-5.5` by default, and the failover phase also requires the expected-offline Worker B row to retain the required capabilities. The consolidated Mac mini final sequence defaults `WORKER_REQUIRED_CAPABILITIES` to `codex-gpt-5.5,gemini-2.5-flash` and requires at least two distinct, real, non-placeholder, non-mock model IDs before prompting for the user token. Use `REQUIRE_DIFFERENT_REGEN_MODEL=0` only for a provisional connectivity run while the second real model is still being configured.
After the named tunnel is running, the quick tunnel is stopped, local proof artifacts are refreshed, handoff bundles are regenerated for the named public URL, and all three production acceptance reports exist, run `make status STATUS_FLAGS=--strict-production`.
