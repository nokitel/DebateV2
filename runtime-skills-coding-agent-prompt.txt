Feature:
Implement runtime skills for Dialectical Engine V2 subagents. A skill is temporary cognitive scaffolding. It does not execute code. It does not grant tools. It does not grant permissions. It shapes subagent identity, reasoning method, search strategy, evidence standards, and constraints.

Critical requirements:
- Skills persist in DB as JSON using existing SkillDefinition.definition.
- When needed, skill JSON must be materialized into a real temporary SKILL.md.
- The temp SKILL.md is mandatory.
- Path format: runtime/skills/{debate_id}/{job_id}/{skill_id}/SKILL.md
- The temp markdown must be deleted after the question/job is fully answered.
- Delete temp markdown on success and failure.
- JSON persists after markdown deletion.
- Skills are under-the-hood only. Users must not see selected skills, skills used, or skill provenance in normal API/UI responses.
- Provenance must still be recorded internally.
- Subagents using these skills must research from official documentation/public primary sources only.
- Code/tool calls are forbidden inside skills.
- Accessing random internet DBs is forbidden.
- Accessing private/non-public data or breaking into databases is forbidden.
- Created skills are usable immediately and reusable immediately for now.
- Retrieval is semantic research/matching for now, but keep it simple and testable.
